# Features Research: A股多智能体量化交易系统

## Table Stakes (必须有)

### 信息收集
- [ ] 多数据源接入 (AKShare)
- [ ] 定时任务调度
- [ ] 数据缓存与更新
- [ ] 数据质量检查

### 决策系统
- [ ] 多Agent投票机制
- [ ] 置信度评级 (A/B/C)
- [ ] 决策日志记录
- [ ] 人工审核确认

### 风控
- [ ] 仓位控制
- [ ] 止损机制
- [ ] 分散度检查

## Differentiators (竞争优势)

### 方差缩小
- [ ] 多源信息聚合
- [ ] 动态权重调整
- [ ] 因子正交化
- [ ] 置信度加权

### Agent专业化
- [ ] 政策解读Agent
- [ ] 行业轮动Agent
- [ ] 资金流向Agent
- [ ] 舆情分析Agent
- [ ] 技术形态Agent

### 量化因子
- [ ] 因子挖掘Agent
- [ ] 因子有效性检验
- [ ] IC/IR分析

## Anti-features (不构建)

- [实盘自动交易] - 仅做信号，模拟盘验证
- [高频/量化杠杆] - 专注短线/中线信号
- [单一Agent决策] - 必须多Agent联合
- [无风控] - 风控Agent有一票否决权

## 复杂度

- 信息收集层: 中等 (10+ Agent)
- 决策层: 中等 (5 Agent + 投票)
- 方差缩小: 高 (多源聚合)
- 风控: 中等 (规则系统)

## 依赖

- Agent框架 → 影响整体架构
- 数据源 → 影响信息收集
- LLM → 影响成本和质量

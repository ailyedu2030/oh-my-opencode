# Architecture: A股多智能体量化交易系统

## 组件架构

```
┌─────────────────────────────────────────────────────────────┐
│                        用户层 (您)                           │
│              决策确认 / 结果审核 / 风险把控                   │
└─────────────────────────────────────────────────────────────┘
                              ↓↑
┌─────────────────────────────────────────────────────────────┐
│                      决策层 (5 Agent)                       │
│   宏观策略 / 行业轮动 / 因子量化 / 情绪资金 / 风控           │
│                         ↓                                   │
│                    投票/加权机制                            │
└─────────────────────────────────────────────────────────────┘
                              ↓↑
┌─────────────────────────────────────────────────────────────┐
│                      分析层 (信号聚合)                       │
│            方差缩小 / 评级计算 / 权重融合                    │
└─────────────────────────────────────────────────────────────┘
                              ↓↑
┌─────────────────────────────────────────────────────────────┐
│                    信息收集层 (10+ Agent)                    │
│   政策 / 行业 / 财报 / 资金 / 舆情 / 技术 / 宏观 / ...      │
└─────────────────────────────────────────────────────────────┘
                              ↓↑
┌─────────────────────────────────────────────────────────────┐
│                       数据层                                 │
│              AKShare / Tushare / 本地缓存                   │
└─────────────────────────────────────────────────────────────┘
```

## 数据流

1. **信息收集**: 10+ Agent 并行获取不同维度信息
2. **信号聚合**: 分析层整合多源信息，方差缩小
3. **联合决策**: 5个决策Agent投票/加权
4. **风控审核**: 风控Agent一票否决
5. **人工确认**: 您最终决策

## 构建顺序

1. 数据层 (AKShare接入)
2. 信息收集层 (2-3个核心Agent)
3. 决策层 (基础投票机制)
4. 风控层
5. 方差缩小
6. 完整Agent生态
